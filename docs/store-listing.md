# Chrome Web Store listing copy

Per-locale store-listing text for the Chrome Web Store Developer Dashboard
(**Store listing** tab → switch *Current editing language*, then paste the
**Description**). The **Summary** is supplied automatically from the package
`_locales` (`__MSG_extDescription__`) and is repeated here only for reference.
The **Title** rows are proposals — see "Localized title" below.

These are transcreations, not literal translations: each is written to read
naturally in its language. Have a native speaker review de / es / fr before
relying on them widely.

## Localized title

The title comes from the package `name`. To localize it per language we would
set `name: "__MSG_extName__"` and add `extName` to each `_locales` file (a code
change shipping in **0.3.6**; not in 0.3.5). All proposals are ≤ 45 characters,
the store title limit.

## English (en)

**Summary:** Privacy-first bilingual web translation that keeps the original text in place so you never lose context.

**Description:**

Margin Read is a privacy-first browser extension for bilingual reading. It leaves the original page untouched and places the translation directly beneath each block of text, so the source and translation sit side by side — no switching tabs, no losing your place, no losing the context.

Why Margin Read

• Private by design. No account, no sign-in, no cloud sync, no telemetry. Only the text you choose is sent — to the provider you pick.

• Use your own provider. Connect OpenAI, Anthropic Claude, or Google Gemini with your own API key, or run a local model (LM Studio, Ollama, llama.cpp, omlx) for fully on-device translation. No vendor lock-in, no built-in quota.

• Reads what matters. It finds paragraphs, headings, lists, and quotes, and skips navigation, buttons, and code — so you translate the article, not the chrome.

• Made for reading. Choose how translations look, keep the original close, and handle dynamic pages and X timelines.

Your data, your provider, your reading — all under your control.

## Japanese (ja)

**Summary:** プライバシー重視の二言語ウェブ翻訳。原文をその場に残すので、文脈を見失いません。

**Description:**

Margin Read は、プライバシーを最優先する二言語リーディング拡張機能です。元のページはそのままに、訳文を各ブロックのすぐ下に表示。原文と訳文を並べて読めるので、タブを切り替える必要も、読んでいた場所を見失うことも、文脈を失うこともありません。

Margin Read が選ばれる理由

• 設計からプライバシー重視。アカウント不要、ログイン不要、クラウド同期なし、テレメトリなし。送信されるのは、あなたが翻訳を選んだテキストだけ。送信先もあなたが指定したプロバイダーです。

• 自分のプロバイダーを使える。OpenAI・Anthropic Claude・Google Gemini を自分の API キーで利用できるほか、LM Studio・Ollama・llama.cpp・omlx でローカルモデルを動かし、完全にデバイス上で翻訳することも可能。ベンダーロックインも、内蔵の利用上限もありません。

• 必要な部分だけ翻訳。段落・見出し・リスト・引用を見つけ出し、ナビゲーションやボタン、コードはスキップ。ページの装飾ではなく、記事そのものを翻訳します。

• 読むためのつくり。訳文の見せ方を選べて、原文をすぐそばに保ち、動的なページや X のタイムラインにも対応します。

あなたのデータ、あなたのプロバイダー、あなたの読書体験 —— すべてをあなたの手に。

## Korean (ko)

**Description:**

Margin Read는 프라이버시를 최우선으로 하는 이중 언어 리딩 확장 프로그램입니다. 원본 페이지는 그대로 두고 번역문을 각 본문 블록 바로 아래에 배치하여, 원문과 번역을 나란히 읽을 수 있습니다. 탭을 전환할 필요도, 읽던 위치를 잃을 일도, 문맥을 놓칠 일도 없습니다.

Margin Read를 선택하는 이유

• 설계부터 프라이버시 중심. 계정 없이, 로그인 없이, 클라우드 동기화 없이, 텔레메트리도 없습니다. 전송되는 것은 사용자가 번역하기로 선택한 텍스트뿐이며, 전송 대상도 사용자가 지정한 제공자입니다.

• 필요한 것만 번역. 문단, 제목, 목록, 인용을 찾아내고 내비게이션, 버튼, 코드는 건너뜁니다. 페이지 장식이 아니라 글 자체를 번역합니다.

• 읽기를 위한 설계. 번역 표시 방식을 선택하고, 원문을 가까이 두며, 동적 페이지와 X 타임라인도 처리합니다.

당신의 데이터, 당신의 제공자, 당신의 읽기 경험 — 모두 당신의 손에.
